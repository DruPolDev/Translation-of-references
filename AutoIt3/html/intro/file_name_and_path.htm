Имя и расположение файла

Каждый раз, когда параметр функции определяется как "имя файла", подразумевается имя файла и полный путь к нему. Хотя функции AutoIt часто принимают относительный путь, при этом корректная работа не гарантируется, поскольку для некоторых Windows API функций требуется полный путь. Если полный путь не указан, а для определения файла используется только имя, AutoIt создас новый файл с этим именем или будет искать в текущем каталоге, путь к которому содержит макрос @WorkingDir.

Рабочий каталог устанавливается в @ScriptDir при запуске скрипта, и это значение можно изменить во время выполнения скрипта, вызвав FileChangeDir. Обратите вниманее на то, что текущий каталог тоже может меняться некоторыми функциями, например, FileOpen/SaveDialog (если успешно выполнится) или Run с параметром "workingdir".

На странице MSDN "Naming Files, Paths, and Namespaces" содержится более полная информация.
