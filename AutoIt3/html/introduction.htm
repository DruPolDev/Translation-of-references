Введение

AutoIt 3 - это свободно распространяемый Basic-подобный скриптовый язык, разработанный для автоматизации управления графическим интерфейсом Windows и создания сценариев общего назначения. В нем используется эмуляция комбинации нажатия клавиш, манипуляции мышью и окнами/элементами управления, для автоматизации задач, способом, который трудно или невозможно использовать в других языках (например, VAScript и SendKeys). К тому же, AutoIt очень компактный, автономный и работает "из коробки" на всех версиях Windows, без необходимости "среды выполнения"!

Первоначально AutoIt разрабатывался для надежного автоматизированного развертывания и настройки тысяч ПК. Со временем он стал мощным языком, поддерживающим комплексные выражения, пользовательские функции, циклы и многое другое, на что могли расчитывать ветераны скриптовых языков:

  • Простой в освоении Basic-подобный синтаксис
  • Эмуляция нажатий клавиш и движений мыши
  • Управление графическими оконными интерфейсами и процессами
  • Взаимодействие со всеми стандартными оконными элементами управления
  • Сценарии могут быть скомпилированы в автономные исполняемые файлы
  • Создание графических интерфейсов пользователя (GUI)
  • Поддержка технологии COM
  • Регулярные выражения
  • Прямой вызов функций внешних DLL и Windows API
  • Скриптовые функции RunAs
  • Подробная справка и большое сообщество на форумах подержки
  • Совместимость с Windows XP SP3/2003 SP2/Vista/2008/Windows 7/2008 R2/Windows 8/2012 R2
  • Поддержка Unicode и x64
  • Цифровая подпись для вашего спокойствия
  • Работа с контролем учетных записей Windows (UAC)

AutoIt разрабатывался максимально компактным, автономным, без внешних .dll файлов и записей в реестре, для безопасного использовния на серверах. Сценарии могут быть скомпилированы в автономные исполняемые файлы с помощью утилиты Aut2Exe.

Кроме того, поставляется комбинированная COM и DLL версия под названием AutoItX, с помощью которой можно добавить уникальные функции AutoIt к вашим любимым языкам программирования!

Самое главное в том, что AutoIt по-прежнему остается свободно распространяемым, но, если вы цените потраченные нами деньги, время и усилия на проект и веб-хостинг, можно сделать пожертвование на домашней странице AutoIt.

Подробнее о возможностях

Basic-подобный синтаксис и богатый набор функций

У AutoIt Basic-подобный синтаксис, значит большинство людей, писавшие когда-либо сценарии или использовавшие языки высокого уровня, смогут легко его понять.

Хотя AutoIt начинал свою жизнь как простой инструмент автоматизации процессов, теперь имеет новые функции и особенности, позволяющие применять его в качестве языка сценариев общего назначения (с потрясающими возможностями в автоматизации!). Особенности языка включают в себя:

  • Джентльменский набор высокоуровневых элементов для функций, циклов и анализа выражений.
  • Огромное количество функций обработки строк и механизм регулярных выражений, совместимый с Perl (с помощью библиотеки PCRE).
  • Поддержка технологии COM.
  • Поддержка вызовов Win32API и внешних DLL.

Встроенный редактор с подсветкой синтаксиса

В комплекте с AutoIt поставляется "облегченная" версия редактора SciTE, упрощающая написание сценариев. Также можно загрузить полную версию редактора, содержащую дополнительные инструменты для работы.

 

Автономность и компактность

AutoIt - это очень компактное и автономное приложение, которое не зависит от среды выполнения, например, .NET или VB. Все, что нужно для работы - это основной исполняемый файл AutoIt (AutoIt3.exe) и сценарий. Сценарий можно преобразовать в автономный исполняемый файл с помощью встроенного компилятора Aut2Exe.

 

Поддержка Unicode и 64-битных систем

AutoIt полностью поддерживает Unicode и включает x64-битные версии всех основных компонентов! О каких еще свободно распространяемых скриптовых языках вы можете сказать такое?

 

Эмуляция нажатий клавиш и движений мыши

Много времени было потрачено на оптимизацию эмуляции нажатия клавиш и движений мыши, для максимально надежной работы во всех версиях Windows. Все операции с мышью и клавиатурой оптимизированы с точки зрения "скорости" эмуляции и функциональности.

 

Управление оконным интерфейсом

Можно перемещать, скрывать, отображать, изменять размер, делать активным, закрывать окна и все, что еще хотите. К оконному интерфейсу можно обращаться по заголовку, тексту в окне, размеру, положению на экране, классу и даже по внутреннему идентификатору Win32 API.

 

Элементы управления

Можно напрямую получать информацию и взаимодействовать с полями ввода, списками, комбинированными элементами, кнопками, строкой состояния, без риска пропустить нажатие клавиши. Возможно работать даже с элементами управления в неактивных окнах!
