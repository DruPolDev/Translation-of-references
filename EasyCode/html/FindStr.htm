FindString method

Prototype

FindString Proto lpszString:LPTSTR, lpszSubstring:LPTSTR, lFlags:DWord (for 32-bit projects)

FindString Proto lpszString:LPTSTR, lpszSubstring:LPTSTR, lFlags:QWord (for 64-bit projects)

Syntax

Invoke FindString, lpszString, lpszSubstring, lFlags

Function

Searches for the specified substring inside the specified string.

Parameters

lpszString

Pointer to the effective address of the null-terminated string where to search.

lpszSubstring

Pointer to the effective address of the null-terminated substring to be searched for.

lFlags

A DWord value (or QWord for 64-bit projects) containing the flags for the search. It can be 0 (no flag), ecMatchCase and/or ecWholeWord and/or ecIgnoreEOL. Any value greater than (ecMatchCase OR ecWholeWord OR ecIgnoreEOL) will be converted to zero (no flag).

Return value

Eax/Rax returns the zero-based position of the substring inside the string, or -1 if the substring was not found.


REMARKS: The FindString method expects lpszString and lpszSubstring to be Unicode strings if the application is running as Unicode, or ANSI strings if not (see the IsAppUnicode method). You can call FindStringA to find substrings inside ANSI strings in Unicode applications, or FindStringW to find substrings inside Unicode strings in ANSI applications.

NOTE: When the ecIgnoreEOL is not specified both characters, NULL (value 0) and VK_RETURN (value 13) will be treated as the end of the string. However, if the ecIgnoreEOL flag is specified, the VK_RETURN character will be ignored and only the NULL character will be treated as the end of the string.
